### 2.3. Листинг и пояснение YAML-файлов

#### 2.3.1. Секреты и ConfigMap (`01-mysql-secret.yaml`)

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: mysql-secret
type: Opaque
data:
  root-password: cm9vdDEyMw==      # "root123" в base64
  user-password: YW5hbHlzdDQ1Ng==   # "analyst456" в base64
***
apiVersion: v1
kind: ConfigMap
metadata:
  name: mysql-config
data:
  database: internet_shop
  user: analyst
```

**Пояснение:** Secret хранит пароли в зашифрованном виде (base64). ConfigMap содержит несекретные параметры — имя базы данных и пользователя. Такой подход соответствует принципам безопасного хранения конфиденциальной информации в Kubernetes.

#### 2.3.2. Развертывание MySQL (`02-mysql-deployment.yaml`)

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: mysql-deployment
spec:
  replicas: 1
  selector:
    matchLabels:
      app: mysql
  template:
    metadata:
      labels:
        app: mysql
    spec:
      containers:
      - name: mysql
        image: mysql:8.0
        ports:
        - containerPort: 3306
        env:
        - name: MYSQL_ROOT_PASSWORD
          valueFrom:
            secretKeyRef:
              name: mysql-secret
              key: root-password
        - name: MYSQL_DATABASE
          valueFrom:
            configMapKeyRef:
              name: mysql-config
              key: database
        - name: MYSQL_USER
          valueFrom:
            configMapKeyRef:
              name: mysql-config
              key: user
        - name: MYSQL_PASSWORD
          valueFrom:
            secretKeyRef:
              name: mysql-secret
              key: user-password
```

**Пояснение:** Deployment описывает запуск одного экземпляра MySQL 8.0. Пароли и настройки передаются через переменные окружения с ссылкой на Secret и ConfigMap. Метка `app: mysql` используется для связывания с сервисом.

#### 2.3.3. Сервис MySQL (`03-mysql-service.yaml`)

```yaml
apiVersion: v1
kind: Service
metadata:
  name: mysql-service
spec:
  selector:
    app: mysql
  ports:
  - port: 3306
    targetPort: 3306
  type: ClusterIP
```

**Пояснение:** Сервис типа ClusterIP создает внутренний сетевой адрес для доступа к MySQL из других подов кластера. Это обеспечивает изоляцию базы данных от внешней сети.

#### 2.3.4. Развертывание phpMyAdmin (`04-phpmyadmin-deployment.yaml`)

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: phpmyadmin-deployment
spec:
  replicas: 1
  selector:
    matchLabels:
      app: phpmyadmin
  template:
    metadata:
      labels:
        app: phpmyadmin
    spec:
      containers:
      - name: phpmyadmin
        image: phpmyadmin/phpmyadmin:latest
        ports:
        - containerPort: 80
        env:
        - name: PMA_HOST
          value: "mysql-service"
        - name: PMA_PORT
          value: "3306"
```

**Пояснение:** phpMyAdmin — веб-интерфейс для управления MySQL. Переменная `PMA_HOST` указывает на имя сервиса MySQL, что позволяет phpMyAdmin автоматически подключиться к базе данных.

#### 2.3.5. Сервис phpMyAdmin (`05-phpmyadmin-service.yaml`)

```yaml
apiVersion: v1
kind: Service
metadata:
  name: phpmyadmin-service
spec:
  selector:
    app: phpmyadmin
  ports:
  - port: 80
    targetPort: 80
    nodePort: 31576
  type: NodePort
```


#### 2.3.6. Job для загрузки данных (`06-loader-job.yaml`)

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: data-loader
spec:
  template:
    spec:
      restartPolicy: Never
      containers:
      - name: loader
        image: data-loader:latest
        imagePullPolicy: Never
        env:
        - name: DB_HOST
          value: "mysql-service"
        - name: DB_USER
          value: "root"
        - name: DB_PASSWORD
          value: "root123"
        - name: DB_NAME
          value: "internet_shop"
```

### 2.4. Сборка Docker-образов

Для работы loader был собран Docker-образ:

```bash
# Переключение на окружение Minikube
eval $(minikube docker-env)

# Сборка образа для загрузчика данных
cd loader
docker build -t data-loader:latest .
```

**Dockerfile для loader:**
```dockerfile
FROM python:3.9-slim
WORKDIR /loader
RUN pip install --no-cache-dir pymysql faker
COPY loader.py .
CMD ["python", "loader.py"]
```

```sql
SELECT p.category,
       SUM(oi.quantity) as items_sold,
       SUM(oi.quantity * oi.price_per_unit) as total_revenue
FROM order_items oi
JOIN products p ON oi.product_id = p.id
GROUP BY p.category
ORDER BY total_revenue DESC;
```
